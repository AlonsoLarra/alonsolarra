4. Explain how routes work in angular and a code example. 10 points.

 If you want to navigate to different pages in your application, but you also want the application to be a SPA (Single Page Application), with no page reloading, you can use the "ngRoute" module. Code Example:

angular
.module('frontpocketApp', [
'ngAnimate',
'ngCookies',
'ngResource',
'ngRoute',
 'ngSanitize',
'ngTouch'
])
.config(function ($routeProvider) {
$routeProvider
.when('/', {
templateUrl: 'views/main.html',
controller: 'MainCtrl',
controllerAs: 'main'
})
.when('/about', {
templateUrl: 'views/about.html',
controller: 'AboutCtrl',
controllerAs: 'about'
})
.when('/tareas', {
templateUrl: 'views/tareas.html',
controller: 'TareasCtrl',
controllerAs: 'tareas'
})
.when('/Assignment1', {
templateUrl: 'views/assignment1.html',
controller: 'Assignment1Ctrl',
controllerAs: 'Assignment1'
})
.otherwise({
redirectTo: '/'
});
});

What this would do internally is that the Browser will load all the information in just one request and as you navigate through the webpage it will change the already loaded information. For example: There is a container that is holding the displayed view, once you request another view, it all remains basically the same but the view without the user noticing that there was not any call to the server.

5. Explain how you can have more than one access key in your computer and how to use each one. Provide code or configuration examples. 10 points.

The following example shows a credentials file with two profiles:

[default]
aws_access_key_id=AKIAIOSFODNN7EXAMPLE
aws_secret_access_key=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
 [user2]
aws_access_key_id=AKIAI44QH8DHBEXAMPLE
aws_secret_access_key=je7MtGbClwBF/2Zp9Utk/h3yCo8nvbEXAMPLEKEY

As you may notice, the difference relies on the [tag].
Here is an example of how you can use your multiple profiles by adding "--profile" and the profile [tag] you want to use:

$ aws ec2 describe-instances --profile user2

If you are going to use a named profile for multiple commands, you can avoid specifying the profile in every command by setting the AWS_DEFAULT_PROFILE environment variable at the command line:

export AWS_DEFAULT_PROFILE=user2
