2. Create a Hello World in NodeJs and create a tutorial about it. 15 points.

 Installing Node.js and NPM is pretty straightforward using Homebrew. Homebrew is a package manager for Mac, it makes installing most open source sofware (like Node) very simple.

First step is to open your terminal and install Homebrew with the following command:

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
Then you need to install Node.

brew install node
Once you have install the node packages, you have to create a javascript file on the same directory of the installed packages, the javascrip file must contain the required code for the "Hello World"; I named my .js file "Hello.js" and here is the code:

Hello.js
var greetings = require("./ExportModule.js");

console.log(greetings.sayHello());

console.log(greetings.diHola());
ExportModule.js
var exports = module.exports = {
sayHello: function() {
return "HELLO WORLD";
},
diHola: function() {
return "Hola mundo";
}
};

 After saving the file, on my Terminal, I positioned the directory on the folder that contain my .js file and wrote:

node hello.js

And that was it, it appears the messages saying 'Hello World' and 'Hola Mundo'.

5.Read and explain the post about Microservices Trade-Offs by Martin Fowler (http://martinfowler.com/articles/microservice-trade-offs.html). 15 points.

"Monoliths and microservices are not a simple binary choice. Both are fuzzy definitions that mean many systems would lie in a blurred boundary area."

 Martin Fowler speaks about some of the advantages of using a microservices architecture like "Independent Deployment" and "Technology Diversity" but also (and mainly) about how this architecture is not always a recipe for success; it has its drawbacks! like the difficulty of maintaining consistency within a distributed system, the need of a mature operations team to manage the variety of the system's services and the complexity of programming distributedly due to the slowness of remote calls and its risk of failure.

Pricing

Google automatically gives you the best price with its Sustained Use Discounts (SUD). The Google SUD discounts the on-demand prices based on how long you use a specific instance type or specific number of CPUs and RAM. If you only use an instance for 25 percent of the billing period, you'll get on-demand pricing. As you use it 50, 75, or 100 percent of the billing period, you’ll get successively higher discounts. With Google, you don’t need to pre-purchase or make decisions because these discounts are applied to your bill so that you automatically get the best price. Source

Compute

Amazon has some impressive high end kit; AWS lets you choose from a range of 40 virtual server instance sizes from low cost ‘nano’ instances to high end memory, storage or GPU optimised products, offering up to 128 ‘vCPU’s and 2TB of memory. Also it offers persistent disk options through Elastic Block Store and comes with a variety of SSD and HDD flavours. Source

Database

With more workloads moving to the cloud the need for NoSQL databases will become ever more important, and again all providers have a good range of options to satisfy most performance/cost requirements. Of all the NoSQL products on offer it’s hard not to be impressed by DocumentDB; once again Microsoft is showing the rest how PaaS should be done. Azure also has the best out-of-the-box support for cross-region geo-replication across its database offerings, without saying that Azure SQL Database is probably the most advanced managed relational database available today.
